cd  <<Название папки >> - перейти в эту папку
dir -посмотреть содержимое папки в которой находишься
git clone << ссылка на репозиторий >>- скачать репозиторий на комп
git status-посмотреть статус репозитория, в котороми сейчас находишься(новые файлы, изменения и тд)
git add << Название файла >>- перевести новый файл в режим готовности к загрузке в репозиторий
git commit –m “Изменения, которые мы внесли(инфа для другого разраба)” –файл переходит в режим полной готовности перед отправкой
git push- команда, отправляющая готовые файлы в репозиторий
git pull- команда для загрузки новых файлов на комп из репозитория
git add –A – команда для того, чтобы закинуть в репозиторий сразу несколько файлов
git – посмотреть полный список команд, существующих в github’е
git remote show origin- посмотреть на какой удаленный репозиторий настроен наш локальный
git branch- посмотреть все существующие ветки и в какой ветке мы находимся в данный момент
git log- посмотреть как делался репозиторий (посмотреть все логи)
git diff- посмотреть все новые изменения
git branch <название ветки>- создать новую ветку с указанным названием
git checkout < название ветки > - перейти в указанную ветку
git merge <название ветки из которой вливаем>- смерджить указанную ветку в ветку, в которой мы сейчас
gitk- с помощью спец. оконной утилиты посмотреть историю веток
git branch -d < название ветки >- удалить указанную ветку
git branch -D < название ветки >- удалить указанную ветку, даже если в ней есть не смёрдженные изменения
